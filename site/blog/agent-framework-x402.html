Semantic Kernel Is Legacy. Agent Framework Gets x402 on Day One.

Microsoft Agent Framework reached Release Candidate four days ago. We shipped the first x402 payment package for it before anyone else. Here's why that matters.

The Shift

Four days ago, Microsoft shipped the Release Candidate for Agent Framework—the unified successor to Semantic Kernel and AutoGen. Semantic Kernel still works and gets 12 months of patches, but the message is clear: new projects should target Agent Framework.

The difference isn't cosmetic. Agent Framework replaces Semantic Kernel's [KernelFunction] plugin system with AIFunctionFactory and Microsoft.Extensions.AI—the same abstraction layer that Azure OpenAI, Ollama, and every other .NET AI provider now implements. It's not just a new API surface. It's the convergence point for the entire .NET AI ecosystem.

When we shipped AgentRails.SemanticKernel.X402 last week, we knew this was coming. So we built the Agent Framework package in parallel and shipped it the day the RC dropped.

Why First Matters

Right now, there are zero x402 payment tools for Microsoft Agent Framework. Not from the x402 community, not from Microsoft, not from any fintech startup. Zero.

The same was true for Semantic Kernel four days ago. We were the first there too. But the window for Semantic Kernel was wide—the framework had been stable for months. The window for Agent Framework is right now, the moment the RC shipped.

If you're a .NET developer starting a new agent project today, you're building on Agent Framework. And when your agent needs to pay for an API, there's exactly one package that does it: 

dotnet add package AgentRails.AgentFramework.X402

What Changed

The core logic is identical. Same wallet, same EIP-3009 signer, same budget management, same thread safety. What changed is the surface:

Semantic Kernel Agent Framework
Tool class X402Plugin X402Tools
Registration [KernelFunction] AIFunctionFactory.Create()
DI extension AddX402Plugin() AddX402Tools()
Dependency Microsoft.SemanticKernel.Abstractions Microsoft.Extensions.AI
Tool exposure SK plugin system IList<AIFunction>

The Microsoft.Extensions.AI dependency is the key difference. It means the x402 tools work with any IChatClient implementation—Azure OpenAI, OpenAI directly, Ollama, local models, anything. You're not locked into one provider.

The Code

Here's what it looks like to give your agent the ability to pay: 

using AgentRails.AgentFramework.X402;
using AgentRails.AgentFramework.X402.Models;
using Microsoft.Extensions.AI;

// Create x402 tools with a $5 budget
var options = new X402PluginOptions
{
    PrivateKey = Environment.GetEnvironmentVariable("WALLET_PRIVATE_KEY")!,
    Network = "eip155:84532",  // Base Sepolia testnet
    BudgetUsd = 5.00m,
};
var wallet = new X402Wallet(options);
var x402 = new X402Tools(wallet, httpFactory, options);

// Register on any IChatClient-based agent
var response = await chatClient.GetResponseAsync(
    "Get the analysis from https://sandbox.agentrails.io/api/x402/protected/analysis",
    new ChatOptions { Tools = x402.GetTools() });

That's it. The agent gets three tools: MakePaidRequestAsync, CheckBudget, and GetPaymentHistory. When it hits an x402-protected API, it automatically detects the 402 response, signs a USDC payment within your budget, and retries. The agent gets the data. The API gets paid.

Three Tools, Not a Framework

We deliberately kept this as a tool library, not a framework. You don't restructure your application around it. You add three functions to your agent's toolbox:

  • MakePaidRequestAsync — HTTP request with automatic 402 payment handling
  • CheckBudget — wallet status, spending, remaining balance
  • GetPaymentHistory — full audit trail of payments made

Budget management is thread-safe. Per-request price caps prevent surprises. AutoPay can be disabled for human-in-the-loop approval. All signing happens locally with Nethereum—private keys never leave your process.

Which Package Should You Use?

Simple decision tree:

  • New project? Use AgentRails.AgentFramework.X402. It's built on the future of .NET AI.
  • Existing Semantic Kernel project? Use AgentRails.SemanticKernel.X402. It works fine and will continue to work. Migrate when you're ready.
  • Python? Use langchain-x402 or crewai-x402.

All four packages share the same x402 V2 protocol implementation, the same EIP-3009 signing, and the same multi-network support (Base, Ethereum, Arc—testnets and mainnets).

What's Next

Agent Framework goes GA in the coming weeks. When it does, every new .NET agent project will start there. The x402 ecosystem needs to be ready for that moment—and now it is.

We're also watching the Agent Framework runtime evolve. Multi-agent orchestration, persistent agents, and agent-to-agent communication are all on Microsoft's roadmap. When agents start calling each other's APIs, the payment layer between them will matter more than ever.

That's what we're building toward: a world where AI agents can autonomously discover, negotiate, and pay for services—no human in the loop, no API keys, no monthly invoices. Just an HTTP request, a USDC signature, and a response.
