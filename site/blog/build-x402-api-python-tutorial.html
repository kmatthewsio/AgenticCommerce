Build Your First x402-Enabled API in 15 Minutes

A step-by-step tutorial: build a FastAPI server that charges USDC per request, then build a LangChain agent that pays for it automatically. Python, two files, 15 minutes.

What You'll Build

By the end of this tutorial, you'll have two things running on your machine:

  1. A FastAPI server that protects an endpoint behind an x402 paywall—$0.01 USDC per request.
  2. A LangChain agent that hits that endpoint, automatically detects the 402 response, pays with USDC, and gets the data.

No dashboard. No API keys. No subscription management. The agent pays per-request with a cryptographic signature, and the server verifies it. That's the entire billing system.

Total time: about 15 minutes. Let's go.

Prerequisites

  • Python 3.10+
  • A Base Sepolia testnet wallet with some test ETH (for gas). You can get test ETH from the Base Sepolia faucet.
  • An OpenAI API key for the agent's LLM.
  • Basic familiarity with FastAPI and LangChain. You don't need to be an expert—the code is short and commented.

Everything runs on Base Sepolia testnet, so you won't spend real money. The USDC is testnet USDC. The ETH is testnet ETH.

Part 1: Build the x402 Server

First, install FastAPI and Uvicorn: 

pip install fastapi uvicorn

Create a file called server.py: 

from fastapi import FastAPI, Request, Response
import json, base64

app = FastAPI()

# x402 pricing: $0.01 per request
PRICE_USD = 0.01
USDC_DECIMALS = 6
PRICE_ATOMIC = int(PRICE_USD * 10**USDC_DECIMALS)  # 10000

PAYMENT_REQUIREMENTS = {
    "x402Version": 2,
    "accepts": [{
        "scheme": "exact",
        "network": "eip155:84532",           # Base Sepolia
        "maxAmountRequired": str(PRICE_ATOMIC),
        "resource": "https://localhost:8000/api/data",
        "description": "Premium market data",
        "mimeType": "application/json",
        "payTo": "0xYOUR_WALLET_ADDRESS",     # Replace with your address
        "maxTimeoutSeconds": 300,
        "asset": "0x036CbD53842c5426634e7929541eC2318f3dCF7e"  # USDC on Base Sepolia
    }]
}

@app.get("/api/data")
async def get_data(request: Request):
    # Check for payment header
    payment = request.headers.get("X-PAYMENT") or \
              request.headers.get("PAYMENT-SIGNATURE")

    if not payment:
        # Return 402 with payment requirements
        encoded = base64.b64encode(
            json.dumps(PAYMENT_REQUIREMENTS).encode()
        ).decode()
        return Response(
            status_code=402,
            headers={"PAYMENT-REQUIRED": encoded},
            content=json.dumps({
                "error": "Payment required",
                "price": f"${PRICE_USD} USDC"
            }),
            media_type="application/json"
        )

    # In production, verify the payment signature here.
    # For this tutorial, we accept any payment header.
    return {
        "data": "Premium market analysis: BTC trending bullish",
        "price_paid": f"${PRICE_USD}"
    }

That's the entire server. Let's break down what it does:

  • Line 7–8: Price is $0.01 USDC. USDC has 6 decimals, so 10000 is the atomic (smallest-unit) representation of one cent.
  • Line 10–22: The PAYMENT_REQUIREMENTS object follows the x402 V2 spec. It tells the client exactly what to pay, on which network, to which address, and for which asset.
  • Line 26–28: We check for the payment header. The x402 spec uses PAYMENT-SIGNATURE; some clients also send X-PAYMENT.
  • Line 30–39: No payment? Return a 402 Payment Required with the requirements base64-encoded in the PAYMENT-REQUIRED header. This is the core of x402: a machine-readable price tag.
  • Line 43–46: Payment present? Return the data. In production, you'd verify the EIP-3009 signature and settle on-chain before responding.

Start the server: 

uvicorn server:app --reload

Test it: 

curl -i http://localhost:8000/api/data

You should see: 

HTTP/1.1 402 Payment Required
payment-required: eyJ4NDAyVmVyc2lvbiI6IDIsIC...
content-type: application/json

{"error": "Payment required", "price": "$0.01 USDC"}

That 402 is the signal. Any x402-aware client—or any LangChain agent with langchain-x402—knows exactly what to do with it.

Part 2: Build the LangChain Agent

Now we build the client: a LangChain agent that can pay for API access autonomously. Install the dependencies: 

pip install langchain-x402 langchain-openai langchain

Create a file called agent.py: 

import os
from langchain_x402 import X402Wallet, X402PaymentTool
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_react_agent
from langchain_core.prompts import ChatPromptTemplate

# 1. Create a wallet with a $5.00 budget
wallet = X402Wallet(
    private_key=os.environ["WALLET_PRIVATE_KEY"],
    network="eip155:84532",   # Base Sepolia
    budget_usd=5.00
)

# 2. Create the x402 payment tool
payment_tool = X402PaymentTool(wallet=wallet)

# 3. Create the agent
llm = ChatOpenAI(model="gpt-4o", temperature=0)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a research agent. Use tools to fetch paid data."),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

agent = create_react_agent(llm, [payment_tool], prompt)
executor = AgentExecutor(agent=agent, tools=[payment_tool], verbose=True)

# 4. Run it
result = executor.invoke({
    "input": "Get the market data from http://localhost:8000/api/data"
})

print(result["output"])

Set your environment variables and run: 

export WALLET_PRIVATE_KEY="0xYOUR_PRIVATE_KEY_HERE"
export OPENAI_API_KEY="sk-YOUR_OPENAI_KEY_HERE"
python agent.py

You'll see the agent's chain-of-thought in the terminal (thanks to verbose=True). It will:

  1. Call the API.
  2. Get the 402 back.
  3. Decide to pay.
  4. Sign the USDC transfer.
  5. Retry with the payment signature.
  6. Get the data.
  7. Return the result.

All of that happened in a few seconds, with zero human intervention.

Part 3: What Just Happened?

Let's walk through the full flow, step by step:

  1. The agent called GET /api/data.
  2. The server returned 402 Payment Required with a price: $0.01 USDC on Base Sepolia.
  3. langchain-x402 detected the 402 and parsed the payment requirements from the PAYMENT-REQUIRED header.
  4. The SDK checked the budget: $5.00 available, $0.01 requested—approved.
  5. The SDK signed an EIP-3009 TransferWithAuthorization—a gasless USDC transfer that authorizes the server to pull exactly $0.01 from the agent's wallet.
  6. The SDK retried the original request, this time with the signed payment in the PAYMENT-SIGNATURE header.
  7. The server verified the signature and returned the data.
  8. The agent received the data and continued its task.

Here's what it looks like as a sequence: 

Agent                        Server
  |                             |
  |--- GET /api/data ---------->|
  |                             |
  |<-- 402 + PAYMENT-REQUIRED --|
  |                             |
  |  (sign EIP-3009 transfer)   |
  |                             |
  |--- GET /api/data ---------->|
  |    + PAYMENT-SIGNATURE      |
  |                             |
  |<-- 200 + data --------------|
  |                             |

Two HTTP requests. One payment. No accounts, no OAuth, no invoices. The price is in the 402 response. The payment is in the retry header. That's the entire protocol.

Why EIP-3009?

EIP-3009 is the key primitive that makes this work at low latency. It's a TransferWithAuthorization—the agent signs a message off-chain that says "I authorize address X to pull Y USDC from my wallet." No on-chain transaction is needed at request time. The server can verify the signature cryptographically in microseconds, then settle on-chain asynchronously.

This means API responses are fast. The agent doesn't wait for a blockchain confirmation. The server knows the payment is valid because the math checks out, and it settles later.

Budget Controls

Notice we set budget_usd=5.00 on the wallet. That's a hard cap. If the agent tries to spend more than $5.00 total, the SDK will refuse the payment before signing anything. You can also set per-request caps. This is how you deploy an agent that can spend money without worrying about runaway costs.

What to Try Next

Hit the Live Sandbox

You don't need to run your own server to test. Point your agent at the AgentRails sandbox—it's a live x402 API that returns real 402 responses: 

result = executor.invoke({
    "input": "Get the analysis from https://sandbox.agentrails.io/api/x402/protected/analysis"
})

No signup. No API key. It just works.

Add Budget Controls and Payment History

The X402PaymentTool exposes methods for checking budget and reviewing past payments: 

# Check remaining budget
print(wallet.remaining_budget)

# Get payment history
for payment in wallet.payment_history:
    print(f"{payment.url} - ${payment.amount_usd} - {payment.timestamp}")

Try CrewAI Instead

If you prefer CrewAI over LangChain, swap the package: 

pip install crewai-x402

The wallet and payment logic are identical. Only the agent framework integration differs.

Build a Multi-Endpoint Server

Different data, different prices. Add more endpoints to your server.py with different price tiers: 

@app.get("/api/basic")
async def basic(request: Request):
    # $0.001 per request
    ...

@app.get("/api/premium")
async def premium(request: Request):
    # $0.01 per request
    ...

@app.get("/api/enterprise")
async def enterprise(request: Request):
    # $0.10 per request
    ...

The agent's SDK reads the price from each 402 response independently. It doesn't care if different endpoints cost different amounts—it checks each one against the budget and pays accordingly.
