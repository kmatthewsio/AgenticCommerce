Your Finance Team Just Got an AI Copilot for x402

We built two Copilot Studio plugins that let your team query x402 revenue, manage agents, and run operations in Microsoft Teams. No code, no dashboards—just ask.

The Problem with Dashboards

You built the dashboard. You wired up the charts. You deployed it at app.agentrails.io with real-time metrics, payment history, network breakdowns, and API key management.

Nobody opens it.

Not because it's bad. Because dashboards require context switching. Your finance lead doesn't want to leave Teams, log into a separate app, navigate to the right page, and interpret a chart. They want to type "What's our x402 revenue this month?" and get an answer.

The best interface is the one your team already uses. For 450 million Microsoft 365 users, that's Teams.

Two OpenAPI specs designed specifically for Microsoft Copilot Studio. Each exposes a curated subset of the AgentRails API as natural-language-callable operations.

FinanceOps: 7 Operations for the Money People

The FinanceOps copilot turns your finance team into power users of the AgentRails API without them knowing what an API is.

Operation What It Does
getPaymentStats Total payments, volume, success rate, per-network breakdown
getPayments Recent payment history with amounts, status, and networks
getPricing Current endpoint prices, supported networks, pay-to address
getAgentDashboard Agent count, active/paused/error breakdown, budget totals

Your CFO types "What's our total x402 revenue?" in Teams. The copilot calls getPaymentStats, gets back the numbers, and answers in natural language. No login, no navigation, no SQL queries.

Agent Executor: 7 Operations for the Builders

The Agent Executor copilot gives your engineering team a conversational interface to the agent management API.

Operation What It Does
listAgents All agents with status, budget, balance, and wallet addresses
createAgent Register a new agent with name, description, and USDC budget
getAgentInfo Detailed info for a specific agent
runAgent Execute a task and get the result, cost, and timing
deleteAgent Remove an agent permanently
getLogs Recent execution logs, filterable by level
getErrorLogs Error-only logs for quick debugging

A developer types "Create a new research agent with a $5 budget" in Teams. The copilot calls createAgent with the right parameters and confirms the agent was created. No curl, no Swagger UI, no terminal.

Setup: 4 Steps, 10 Minutes

You don't write code. You don't deploy anything. You upload a file and configure a key.

  1. Upload the spec. In Copilot Studio, go to Actions → Add an action → OpenAPI. Upload the .swagger.json file.
  2. Configure auth. Set authentication type to API Key, header name to X-API-Key, paste your AgentRails API key.
  3. Select operations. Choose which of the 7 operations to enable. Start with a few, add more later.
  4. Publish. Deploy to Teams. Your team can start asking questions immediately.

That's it. The copilot handles intent detection, parameter extraction, API calls, and response formatting. You just gave it the swagger file and a key.

Why Swagger 2.0?

Copilot Studio has specific requirements that constrain the spec format:

  • Swagger 2.0 JSON only. No OpenAPI v3, no YAML. Copilot Studio's connector parser only supports the older format.
  • Flat schemas. No nested objects, no oneOf/allOf/anyOf, no circular $ref. The LLM handles flat key-value responses much better.
  • Rich descriptions. Every operation, parameter, and response property has a description written for the LLM. These aren't developer docs—they're prompts that tell the copilot when and how to call each endpoint.
  • 7 operations per spec. Quality degrades above ~10 operations. Two focused specs beat one bloated one.

We tested all 14 operations against the live api.agentrails.io API before publishing. Every endpoint returns 200, every response shape matches the schema.
