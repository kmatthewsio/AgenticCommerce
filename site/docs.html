3

Hit a Protected Endpoint

Call a paid endpoint without paying â€” you'll get a 402 Payment Required response:

curl -i https://sandbox.agentrails.io/api/x402/protected/analysis

The response includes a PAYMENT-REQUIRED header with the amount, recipient, and supported networks. In production, your agent reads this and signs a payment automatically.

4

Create an Agent

Register an AI agent on the platform. Requires your API key:

curl -X POST https://sandbox.agentrails.io/api/agents \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ar_test_YOUR_KEY_HERE" \
  -d '{"name": "My Test Agent", "description": "Testing the API"}'

Replace ar_test_YOUR_KEY_HERE with the API key from Step 1.

5

Execute a Test Payment

Simulate a payment in sandbox (no real funds). The amount is passed as a query parameter:

curl -X POST "https://sandbox.agentrails.io/api/x402/test/execute-payment?amountUsdc=0.01" \
  -H "X-API-Key: ar_test_YOUR_KEY_HERE"

In production, your agent signs an EIP-3009 authorization and includes it in the PAYMENT-SIGNATURE header instead.

6

Check Payment History

See all payments recorded against your account:

curl https://sandbox.agentrails.io/api/x402/payments \
  -H "X-API-Key: ar_test_YOUR_KEY_HERE"

Returns an array of payment objects with amount, status, network, and timestamps.

7

Check Stats

Get aggregate stats for your payment activity:

curl https://sandbox.agentrails.io/api/x402/stats \
  -H "X-API-Key: ar_test_YOUR_KEY_HERE"

Returns total payments, volume, success rate, and per-network breakdowns.

8

Log into the Dashboard

See everything in a web UI. Log in with the same email and password you used in Step 1:

open https://app.agentrails.io

The dashboard shows your agents, payments, stats, and API keys â€” all synced across sandbox and production.

How x402 Works

x402 is an open protocol that enables HTTP-native payments. It uses the HTTP 402 status code to signal that payment is required, along with machine-readable headers that tell the client exactly how to pay.

x402 is the payment layer that makes AI agents commercially autonomous. While governance controls what agents can do, x402 enables them to pay for APIs, data, and compute without human intervention.

The Payment Flow

1

Client requests protected resource

Standard HTTP GET/POST to your API endpoint

2

Server returns 402 + payment requirements

PAYMENT-REQUIRED header contains: amount, recipient, network, asset

3

Client signs EIP-3009 payment authorization

Cryptographic signature authorizing USDC transfer (gasless for the payer)

4

Client retries with payment proof

PAYMENT-SIGNATURE header contains signed authorization

5

Server verifies, settles, and returns data

Payment settles on-chain, response includes PAYMENT-RESPONSE with transaction hash

Why EIP-3009?

EIP-3009 allows "transferWithAuthorization" - a gasless way to authorize USDC transfers. The payer signs a message, and the recipient (or a facilitator) submits the transaction. This means AI agents don't need ETH for gas - they just need USDC.

Server Integration

Add x402 payment protection to your API endpoints. We provide middleware for ASP.NET Core, with more frameworks coming soon.

ASP.NET Core

Add the [X402Payment] attribute to any endpoint:

[X402Payment(0.01, Description = "Premium API call")]
[HttpGet("premium")]
public IActionResult Premium()
{
    // Payment already verified by middleware
    var payer = HttpContext.GetX402Payer();
    var txHash = HttpContext.GetX402TransactionHash();

    return Ok(new {
        data = "Premium content",
        payer,
        txHash
    });
}

The middleware automatically handles 402 responses, payment verification, and on-chain settlement.

LangChain Integration

from langchain_x402 import X402Wallet, X402PaymentTool
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_react_agent

# 1. Create a wallet with a USDC budget
wallet = X402Wallet(
    private_key=os.environ["WALLET_PRIVATE_KEY"],
    network="eip155:8453",  # Base mainnet
    budget_usd=10.00
)

# 2. Create the payment tool
tool = X402PaymentTool(wallet=wallet)

# 3. Add to your agent
llm = ChatOpenAI(model="gpt-4o")
agent = create_react_agent(llm, tools=[tool], prompt=your_prompt)
executor = AgentExecutor(agent=agent, tools=[tool])

# 4. Agent can now access any x402-enabled API
result = executor.invoke({
    "input": "Get data from https://api.example.com/paid-endpoint"
})

Semantic Kernel Integration .NET

using AgentRails.SemanticKernel.X402;
using AgentRails.SemanticKernel.X402.Extensions;
using Microsoft.SemanticKernel;

var builder = Kernel.CreateBuilder();
builder.AddOpenAIChatCompletion("gpt-4o", apiKey);

// 1. Register the x402 plugin

    options.PrivateKey = Environment.GetEnvironmentVariable("WALLET_PRIVATE_KEY")!;
    options.Network = "eip155:84532";  // Base Sepolia
    options.BudgetUsd = 5.00m;
});
builder.Plugins.AddFromType<X402Plugin>();

var kernel = builder.Build();

// 2. Agent can now access any x402-enabled API
var result = await kernel.InvokePromptAsync(
    "Get the analysis from https://sandbox.agentrails.io/api/x402/protected/analysis");

How it works

When your agent requests a paid resource, the SDK automatically detects the 402 response, signs a USDC payment authorization, and retries the request. Your agent gets the data, the API gets paid. No API keys, no subscriptions. Works with Python (LangChain, CrewAI) and .NET (Semantic Kernel, Agent Framework).

MCP Servers

The Model Context Protocol (MCP) is an open standard that lets AI agents securely access external data sources, tools, and APIs through a unified interface. We build custom MCP servers that connect your Copilot Studio agents to internal systems â€” with enterprise-grade security and scoping.

What is MCP?

The Model Context Protocol (MCP) standardizes how AI models interact with external data and tools. Instead of building custom integrations for each data source, MCP provides a single protocol that any AI agent can use. Think of it as a universal adapter between AI agents and your enterprise systems.

Internal Data Access

Connect agents to SharePoint, Dataverse, SQL databases, and internal APIs through secure MCP endpoints. Agents query data without direct database access.

Tool Orchestration

Expose internal tools â€” approval workflows, ticket creation, report generation â€” as MCP tools that agents can invoke within defined guardrails.

Security & Scoping

Every MCP server enforces row-level security, column-level masking, and role-based access. Agents see only what their permissions allow.

Trust API

Before paying a service, verify its trustworthiness. The Trust API combines service registry data with payment history to generate trust scores.

Check Service Trust

Get a trust score before making a payment:

curl "https://sandbox.agentrails.io/api/trust/check?serviceUrl=https://api.example.com/data"
{
  "serviceUrl": "https://api.example.com/data",
  "registered": true,
  "verified": true,
  "name": "Example Data API",
  "stats": {
    "totalPayments": 1247,
    "settledPayments": 1223,
    "successRate": 98.1,
    "totalVolumeUsdc": 12.47
  },
  "trustScore": "high"
}

Trust Score Levels

high Verified + 100+ settled payments + 95%+ success rate
medium Registered + 50+ settled payments + 90%+ success rate
low Registered or 10+ payments but doesn't meet higher criteria
unknown Not registered and fewer than 10 payments

Register Your Service

Make your x402 service discoverable:

curl -X POST "https://sandbox.agentrails.io/api/trust/register" \
  -H "Content-Type: application/json" \
  -d '{
    "serviceUrl": "https://your-api.com/endpoint",
    "name": "Your API Name",
    "description": "What your API does",
    "ownerWallet": "0x...",
    "priceUsdc": 0.01
  }'

Browse Registry

List all registered services:

curl "https://sandbox.agentrails.io/api/trust/registry?verified=true"

Why Trust Matters

EIP-3009 signatures prove payment authorization, but not service quality. The Trust API provides a lightweight layer for agents to verify services before paying - combining registry data with on-chain payment history for reliable trust signals.

Billing API

Track usage and billing for pay-as-you-go accounts. Usage is metered and billed monthly via Stripe.

Get Usage (with API Key)

Authenticate with your API key to get usage summary:

curl "https://api.agentrails.io/api/billing/usage?days=30" \
  -H "X-Api-Key: ar_live_your_key_here"
{
  "organization": {"id": "...", "tier": "payg"},

  "usage": {
    "transactionCount": 142,
    "totalVolumeUsdc": 284.50,
    "totalFeesUsd": 1.42,
    "billedFeesUsd": 0.00,
    "unbilledFeesUsd": 1.42

Get Usage by Email

Check usage for a specific account. Requires API key auth:

curl "https://api.agentrails.io/api/billing/usage/developer@example.com?days=30" \
  -H "X-API-Key: ar_live_your_key_here"

How Billing Works

Pay-as-you-go accounts are charged 0.5% of transaction volume. Usage is tracked per transaction and aggregated monthly. At the start of each month, unbilled usage is reported to Stripe and invoiced to your card on file. Pro tier ($49/mo) includes unlimited transactions with 0% fees.
